Create setup.sh at repo root. The script must follow this exact 9-step structure from 07-RESEARCH.md:

**Header:**
```bash
#!/usr/bin/env bash
set -euo pipefail

# ESP32 MicroPython Dev Station — Setup Script
# Installs everything needed on a fresh Raspberry Pi (or any Linux host).
# Safe to re-run: each step checks before acting.
#
# Usage: bash setup.sh
#        curl -fsSL https://raw.githubusercontent.com/bakedpanda/ESP32-server/main/setup.sh | bash

REPO_URL="https://github.com/bakedpanda/ESP32-server.git"
INSTALL_DIR="$HOME/esp32-station"
SERVICE_NAME="esp32-station"
CREDS_PATH="/etc/esp32-station/wifi.json"

log()  { echo "[setup] $*"; }
warn() { echo "[setup] WARNING: $*" >&2; }
die()  { echo "[setup] ERROR: $*" >&2; exit 1; }
```

**Step 1 — Pre-flight checks** (before doing anything mutating):
- Check that git is available: `command -v git >/dev/null 2>&1 || die "git not found. Install with: sudo apt install git"`
- Check that python3-venv is available: `python3 -c "import venv" 2>/dev/null || die "python3-venv not available. Install with: sudo apt install python3-venv"`

**Step 2 — Clone or update repo (idempotent):**
```bash
if [ -d "$INSTALL_DIR/.git" ]; then
    log "Repository already cloned — pulling latest..."
    git -C "$INSTALL_DIR" pull
else
    log "Cloning repository to $INSTALL_DIR..."
    git clone "$REPO_URL" "$INSTALL_DIR"
fi
```

**Step 3 — Create virtualenv (idempotent):**
```bash
VENV="$INSTALL_DIR/venv"
if [ ! -f "$VENV/bin/python3" ]; then
    log "Creating virtualenv..."
    python3 -m venv "$VENV"
else
    log "Virtualenv already exists — skipping creation."
fi
log "Installing Python dependencies..."
"$VENV/bin/pip" install --quiet -r "$INSTALL_DIR/requirements.txt"
```

**Step 4 — Add user to dialout group (idempotent):**
Note: `set -e` will exit if `grep -q` returns 1. Wrap in `if` to avoid this (Pitfall 4):
```bash
if groups "$USER" | grep -q '\bdialout\b'; then
    log "User $USER already in dialout group."
else
    log "Adding $USER to dialout group (requires sudo)..."
    sudo usermod -aG dialout "$USER"
    warn "You must log out and back in (or reboot) for dialout group membership to take effect."
fi
```

**Step 5 — Prompt for WiFi credentials (idempotent — check if file exists first, Pitfall 5):**
```bash
SKIP_CREDS=0
if [ -f "$CREDS_PATH" ]; then
    log "Credentials file already exists at $CREDS_PATH."
    read -rp "Overwrite existing credentials? [y/N]: " OVERWRITE
    [[ "$OVERWRITE" =~ ^[Yy]$ ]] || SKIP_CREDS=1
fi

if [ "$SKIP_CREDS" -eq 0 ]; then
    echo ""
    echo "Enter WiFi and WebREPL credentials (will be written to $CREDS_PATH)."
    read -rp  "WiFi SSID: " WIFI_SSID
    read -rsp "WiFi password: " WIFI_PASSWORD
    echo
    read -rsp "WebREPL password (for over-WiFi REPL access): " WEBREPL_PASSWORD
    echo
fi
```

**Step 6 — Write credentials file (safe for special characters, Pitfall 3):**
Use the Python one-liner from RESEARCH.md which is safe against `%`, `"`, and `\` in credentials:
```bash
if [ "$SKIP_CREDS" -eq 0 ]; then
    log "Writing credentials to $CREDS_PATH (requires sudo)..."
    sudo mkdir -p /etc/esp32-station
    "$VENV/bin/python3" -c "
import json, sys
data = {'ssid': sys.argv[1], 'password': sys.argv[2], 'webrepl_password': sys.argv[3]}
print(json.dumps(data))
" "$WIFI_SSID" "$WIFI_PASSWORD" "$WEBREPL_PASSWORD" | sudo tee "$CREDS_PATH" > /dev/null
    sudo chmod 600 "$CREDS_PATH"
    log "Credentials written. File permissions set to 600."
    # Clear credential variables from environment immediately
    unset WIFI_SSID WIFI_PASSWORD WEBREPL_PASSWORD
fi
```

**Step 7 — Patch and install systemd service (idempotent via `restart`):**
This is the critical step. The service file hardcodes User=esp32 and /home/esp32 — MUST patch with sed before installing:
```bash
SERVICE_SRC="$INSTALL_DIR/esp32-station.service"
SERVICE_DST="/etc/systemd/system/esp32-station.service"

log "Installing systemd service (requires sudo)..."
sed -e "s|User=esp32|User=${USER}|g" \
    -e "s|/home/esp32|${HOME}|g" \
    "$SERVICE_SRC" | sudo tee "$SERVICE_DST" > /dev/null

sudo systemctl daemon-reload
sudo systemctl enable "$SERVICE_NAME"
sudo systemctl restart "$SERVICE_NAME"
log "Service installed, enabled, and started."
```

**Step 8 — Verify service is running:**
```bash
if ! sudo systemctl is-active --quiet "$SERVICE_NAME"; then
    warn "Service did not start. Check logs with: journalctl -u $SERVICE_NAME -n 20"
else
    log "Service is active."
fi
```

**Step 9 — Print endpoint URL and claude mcp add command (D-07):**

```bash
HOSTNAME=$(hostname)
echo ""
echo "============================================================"
echo " Setup complete!"
echo "============================================================"
echo ""
echo " MCP server running at:"

echo ""
echo " If hostname resolution fails, replace ${HOSTNAME}.local with the Pi's IP address."
echo " Find it with: hostname -I"
echo ""
echo " Next: restart your terminal session (for dialout group to take effect)."
echo "============================================================"
```
