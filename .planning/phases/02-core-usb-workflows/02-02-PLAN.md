    - Run:

    FUNCTION: read_serial(port: str, timeout: int = READ_SERIAL_TIMEOUT) -> dict
    - Use mpremote to capture recent serial output. Since mpremote has no dedicated "read" subcommand,
      execute a no-op and capture stdout (any buffered output from the board comes through):
        subprocess.run(
            [MPREMOTE_CMD, "connect", port, "exec", ""],
            capture_output=True, text=True, timeout=timeout
        )
    - Wrap in try/except subprocess.TimeoutExpired:
        return {"error": "read_timeout", "detail": f"serial read timed out after {timeout}s"}
    - If returncode != 0:
        return {"error": "read_failed", "detail": result.stderr.strip()}
    - Return: {"port": port, "output": result.stdout}   # preserve newlines for serial output

    FUNCTION: soft_reset(port: str) -> dict
    - Run: exec_repl(port, "import machine; machine.soft_reset()", timeout=10)
    - Note: soft_reset() may cause the subprocess to exit with a non-zero code because the board resets
      mid-execution. Treat returncode 0 OR non-zero-with-empty-stderr as success (board reset successfully).
    - Implementation:
        result = subprocess.run(
            [MPREMOTE_CMD, "connect", port, "exec", "import machine; machine.soft_reset()"],
            capture_output=True, text=True, timeout=10
        )
        # Board disconnects after reset — non-zero exit is expected; treat empty stderr as success
        if result.returncode != 0 and result.stderr.strip():
            return {"error": "soft_reset_failed", "detail": result.stderr.strip()}
        return {"port": port, "reset": "soft"}

    FUNCTION: hard_reset(port: str) -> dict

        )
        if result.returncode != 0 and result.stderr.strip():
            return {"error": "hard_reset_failed", "detail": result.stderr.strip()}
        return {"port": port, "reset": "hard"}
