    Create `tools/webrepl_cmd.py`:
    - Constants: WEBREPL_PORT = 8266
    - Function: `webrepl_exec(host: str, password: str, command: str, timeout: int = 15, port: int = WEBREPL_PORT) -> dict`
    - Uses Python stdlib `socket` module to implement websocket handshake (matching vendored webrepl_cli.py pattern)
    - Websocket handshake: send GET / HTTP/1.1 with Upgrade: websocket headers, read until blank line
    - Login: read data until "Password:" prompt appears, send password as text frame
    - Command execution: enter raw REPL mode (send \x01 = Ctrl-A), send command + \x04 (Ctrl-D) to execute, read output until ">" prompt or "OK" marker
    - Return {"output": stripped_output} on success
    - Return {"error": "wifi_timeout", "detail": f"WebREPL connection to {host} timed out after {timeout}s"} on socket.timeout
    - Return {"error": "wifi_unreachable", "detail": f"WebREPL connection to {host} failed: {e}"} on OSError
    - Return {"error": "webrepl_exec_failed", "detail": str(e)} on any other Exception
    - If password is None or empty, return {"error": "invalid_params", "detail": "password required for WiFi command execution"}
    - Never raise to callers
    - Include internal helper functions for websocket frame read/write (text frames only, opcode 0x81)

    Create `tools/board_status.py`:
    - Import exec_repl from tools.repl
    - Import webrepl_exec from tools.webrepl_cmd
    - Import json from stdlib
    - Constants:
      - STATUS_SCRIPT: MicroPython snippet (single string) that imports sys, gc, json, tries `import os` then `import uos as os`, tries `import network` with try/except, calls gc.collect(), reads os.statvfs('/'), tries network.WLAN(network.STA_IF).config('dhcp_hostname') for hostname, prints json.dumps() with keys: firmware, wifi_connected, ip_address, free_memory, free_storage, board. Per D-01 and D-03.
      - HEALTH_PING = "print(1)"
      - HEALTH_TIMEOUT = 5
    - Function: `get_status(port=None, host=None, password=None) -> dict`
      - Per D-02 and D-13: exactly one of port/host required
      - If both provided: return {"error": "invalid_params", "detail": "Provide either port or host, not both"}
      - If neither provided: return {"error": "invalid_params", "detail": "Provide either port or host"}
      - USB path (port set): call exec_repl(port, STATUS_SCRIPT, timeout=10)
      - WiFi path (host set): validate password not None, call webrepl_exec(host, password, STATUS_SCRIPT, timeout=15)
      - If result has "error" key, return it directly
      - Parse result["output"] as JSON, add "transport" key ("usb" or "wifi"), return the dict
      - On json.JSONDecodeError or KeyError: return {"error": "status_parse_failed", "detail": f"Could not parse board output: {result.get('output', '')}"}
    - Function: `check_health(port=None, host=None, password=None) -> dict`
      - Per D-04, D-05, D-06: three failure states
      - Same port/host validation as get_status
      - USB path: first check port exists via serial.tools.list_ports.comports() -- if port not in [p.device for p in comports()], return {"status": "not_found", "detail": f"No device at {port}"}
      - USB path: call exec_repl(port, HEALTH_PING, timeout=HEALTH_TIMEOUT)
      - WiFi path: call webrepl_exec(host, password, HEALTH_PING, timeout=HEALTH_TIMEOUT)
      - If result has "error" and "timeout" is in result["error"]: return {"status": "unresponsive", "detail": result["detail"]}
      - If result has "error" (non-timeout): return {"status": "not_found", "detail": result["detail"]}
      - Otherwise: return {"status": "healthy"}

    Run tests -- they must all PASS.
